第 1 课：Python 基础语法核心（工程师版）#

1️⃣ 变量与动态类型系统 2️⃣ 基本数据类型 3️⃣ None / True / False 的本质

一、变量与动态类型系统（Python vs Go）#

1. Python 是强类型 + 动态类型#

a = 10
a = "hello"
a = [1, 2, 3]

变量 不是类型容器，而是引用

👉 类似 Go：

var a interface{}

2. Python 变量本质是：指针 + 对象#

a = 10
b = a

内存模型：

a ──▶ PyObject(10)
b ──▶ PyObject(10)

验证：

a = 10
b = a
print(id(a), id(b))   # 一样

id() = 对象的内存地址（CPython 中）

3. 赋值不是拷贝，而是引用绑定#

a = [1, 2, 3]
b = a
b.append(4)
print(a)

输出：

[1, 2, 3, 4]

本质：

a ──▶ list
b ──▶ same list

对比 Go：

slice := []int{1,2,3}
s2 := slice

行为极其相似

二、基本数据类型#

Python 数据类型 = 6 大类

分类 类型
数值 int, float, complex
布尔 bool
空值 None
字符串 str
容器 list, tuple, set, dict
进阶 bytes, bytearray

1. 不可变类型 vs 可变类型（极重要）#

不可变 可变
int list
float dict
bool set
str bytearray
tuple

👉 不可变对象：内容不可修改，只能新建

a = 10
a += 1   # 不是原地修改，而是创建新对象

2. 可变对象的“原地修改”#

lst = [1, 2, 3]
lst.append(4)

不会创建新对象

这直接影响：

  • 并发安全
  • 参数传递
  • 函数副作用
  • 缓存设计

3. Python 参数传递本质#

永远是：对象引用传递（call by object reference）

def foo(x):
    x.append(4)

a = [1,2,3]
foo(a)
print(a)

输出：

[1,2,3,4]

你可以直接类比 Go 的 slice 传参

三、None / True / False 的本质#

1. None 是一个真实对象#

None is None   # True

type(None)
# <class 'NoneType'>

只有一个实例：单例

2. True / False 也是对象#

type(True)
# <class 'bool'>

True + 1   # 2
False + 1  # 1

因为：

bool 是 int 的子类

验证：

isinstance(True, int)  # True

3. Python 的真假判断规则#

bool(0)        -> False
bool("")       -> False
bool([])       -> False
bool(None)     -> False

空 = False

这决定了：

if data:

等价于：

if data is not None and len(data) > 0

Python = 一切皆对象 + 引用传递 + 动态类型 + 强类型

🧠 第 2 课：dict & set —— Python 的灵魂#

函数传参 tuple list set dict#

如果你真正理解了 dict，你就真正理解了 Python。

本节 3 个核心知识点：

1️⃣ dict 底层结构 & 哈希表原理 2️⃣ Python 的 hash / eq 机制 3️⃣ set 本质 & 高性能根源

一、dict 是 Python 的“第一公民”#

Python 里：

  • 变量
  • 对象
  • 模块
  • 类
  • 实例
  • kwargs
  • globals
  • locals

👉 本质全是 dict

例如：

class A:
    x = 10

底层等价：

A.__dict__ = {"x": 10}

二、dict 底层结构（哈希表）#

Python dict 是：

开放寻址哈希表（open addressing hash table）

结构模型：

hash(key) → index → table[index]

核心特性：#

  • 查找：O(1)
  • 插入：O(1)
  • 删除：O(1)

三、hash & eq：dict 的双核心机制#

dict 判断两个 key 是否相同，必须同时满足：

hash(a) == hash(b)
and
a == b

1️⃣ hash 决定：桶位置#

hash(1)
hash("abc")
hash((1,2,3))

2️⃣ eq 决定：逻辑相等#

1 == True   # True

四、经典灵魂题拆解#

d = {}
d[1] = "a"
d[True] = "b"
print(d)

你刚才已经推过类似的，我们现在完整拆解

Step 1：计算 hash#

hash(1) == hash(True)

结果：

True

因为：

bool 是 int 子类
True == 1

Step 2：判断 eq#

1 == True   # True

结论：#

key 冲突 → 覆盖

所以：

{1: "b"}

五、这道题背后牵出 Python 三大核心设计#

1️⃣ bool 继承自 int 2️⃣ hash & eq 联合判断 3️⃣ dict 冲突覆盖机制

六、你现在的 Go 对照模型#

Go：

map[int]string

Python：

dict[int] = value

区别：

维度 Go Python
key 可变 ❌ ❌
hash 可自定义 ❌ ✅
eq 可自定义 ❌ ✅
冲突策略 链表/红黑树 开放寻址

七、set 本质是什么？#

一句话：

set = 只有 key，没有 value 的 dict

s = {1,2,3}

底层本质：

{1: None, 2: None, 3: None}

八、为什么 set 查找极快？#

x in set_obj

→ hash 定位 → O(1)

而：

x in list_obj

→ 线性扫描 → O(n)

九、为什么 list 不能做 dict key？#

因为：

dict key 必须 hash 值稳定

而：

  • list → 可变 → hash 不稳定 ❌
  • tuple → 不可变 → hash 稳定 ✅

十、你现在的理解层级#

你已经具备：

👉 dict + set + hash + eq + 引用模型 + 运算符模型

这已经是 Python 工程核心区

    def __eq__(self, other):
        return True
